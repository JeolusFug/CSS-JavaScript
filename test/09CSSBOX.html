레이아웃을 구성하는 CSS 박스 모델

블럭 레벨 요소는 글 자체가 브라우저 상에서 한 줄을 차지하는 것


인라인 레벨 요소는 자기가 딱 필요한 영역만을 차지하는 것


p 태그를 사용하면 블럭 레벨, 한줄


span 태그를 사용하면 인라인 레벨, 필요한 영역


이게 박스모델


콘텐츠 영역, 테두리, 패딩등 모든 값을 잘 생각해서 설정해야함


아니면 넘쳐서 보이지 않거나 이상하게 보일 수 있음


box-sizing: border-box 를 사용하면 테두리 굵기도 포함해서 계산해달라는 뜻


박스 모델의 값 지정 방향은 top right bottom left 시계방향으로 돌아감


border-style 속성은 기본 none으로 표시가 안되고


테두리를 그리려면 맨 먼저 테두리 스타일부터 지정해야 한다


none 테두리 없음. 기본값


hidden 테두리를 감춤 border-collapse: collapse(테두리 합치기) 일경우 다른 테두리도 감춤


solid 실선


dotted 점선


dashed 짧은 직선


double 이중선 두 선 사이의 간격이 border-width가 된다


groove 표에서 테두리를 창에 조각한듯 홈이 파인듯 입체 느낌


inset 표에서 border-collapse: seperate일 경우 전체 박스 테두리가 창에서 튀어나온것처럼 표시된다.


inset 표에서 border-collapse: collapse일 경우 ridge와 똑같이 표시된다.


ridge 테두리를 창에서 튀어나온것처럼 표시한다


border-width, border-color 도 설정가능 하지만 동서남북 각각 따로 지정할 경우 border-style을 가장 먼저 지정해줘야함


border-width는 px, 혹은 thin->medium->thick 굵기 설정 가능


border-radius는 꼭지점을 둥글게 처리. 이미지도 가능


마진을 설정할 때 마진 중첩 현상을 조심해야함


마진을 세로로 배치할 때 마진과 마진이 만나면 마진이 두배가 되지는 않음


위의 박스의 아래마진을 30px, 밑의 박스의 위마진을 30px로 설정하면


60px이 될것같지만 30px 밖에 되지 않음


50px, 30px이면 50px이 됨 둘중 큰 마진으로 겹쳐진다


가로로 배치될 때는 상관이 없음


padding은 콘텐츠와 밖의 테두리 사이의 여백


패딩을 지정하지 않으면 바짝 붙어 보기 좋지 않음


패딩을 지정하는 것은 마진과 똑같음
